# Contributing

Contributions are welcome. Please keep changes focused and minimal.

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b fix/your-fix` or `feat/your-feature`
3. Make your changes
4. Commit with a clear message: `git commit -m "fix: describe what changed"`
5. Push and open a Pull Request

## What to Contribute

- Bug fixes in skill rules or templates
- New example reports (`examples/`)
- Improvements to the quality gate checklist
- Additional reference guides
- Template variants (e.g., journal format, extended abstract)

## What NOT to Contribute

- Project-specific content (university names, course codes, etc.)
- Changes that break the zero-code-artifact rule
- Hardcoded language assumptions

## Reporting Issues

Open a [GitHub Issue](https://github.com/EnesDemir143/awesome-ieee-report/issues) with:
- What you expected
- What actually happened
- Minimal example if applicable
